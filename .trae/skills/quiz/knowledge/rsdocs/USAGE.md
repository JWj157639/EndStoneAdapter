# 使用说明

## 命令行参数详解

### 基本语法格式

```batch
console_auto_restart.exe <进程名> <可执行文件路径> [选项]
```

### 必需参数说明

| 参数 | 说明 | 示例 |
|------|------|------|
| 进程名 | 要监控的进程名称（如：notepad.exe） | `notepad.exe` |
| 可执行文件路径 | 要重启的可执行文件完整路径 | `"C:\Windows\System32\notepad.exe"` |

### 可选参数配置

| 选项 | 参数类型 | 说明 | 默认值 | 示例 |
|------|----------|------|--------|------|
| -d | 毫秒 | 重启延迟时间 | 3000 | `-d 5000` |
| -c | 毫秒 | 检测间隔时间 | 5000 | `-c 3000` |
| -m | 次数 | 最大重启次数 | 10 | `-m 5` |
| -l | 等级 | 日志等级（debug/info/warn/error/fatal） | info | `-l debug` |
| -h | - | 显示帮助信息 | - | `-h` |

## 使用示例合集

### 示例 1：监控记事本

```batch
console_auto_restart.exe notepad.exe "C:\Windows\System32\notepad.exe"
```

### 示例 2：自定义参数

```batch
console_auto_restart.exe myapp.exe "C:\Program Files\MyApp\myapp.exe" -d 5000 -c 3000 -m 5
```

参数说明：
- 重启延迟：5000毫秒（5秒）
- 检测间隔：3000毫秒（3秒）
- 最大重启次数：5次

### 示例 3：调试模式

```batch
console_auto_restart.exe myapp.exe "C:\MyApp\myapp.exe" -l debug -d 2000
```

参数说明：
- 日志等级设置为 DEBUG
- 重启延迟：2000毫秒（2秒）

## MQTT远程控制功能（可选）

如果程序编译时定义了 `PAHO_MQTT_ENABLED` 宏并包含了MQTT功能，将自动连接到MQTT服务器并支持远程控制。

### MQTT主题配置

| 主题 | 说明 | 用法 |
|------|------|------|
| restart/control/{进程名} | 接收控制指令 | 用于发送远程指令 |
| restart/status/{进程名} | 发布状态信息 | 用于报告当前状态 |

### MQTT控制指令列表

| 消息内容 | 说明 |
|----------|------|
| RESTART | 手动触发重启 |
| STOP | 停止监控 |
| START | 开始监控 |
| STATUS | 请求状态信息 |

### MQTT连接配置详情

程序默认连接配置：
- 服务器地址：localhost:1883
- 客户端ID：`restart_tool_{进程名}`

如果MQTT连接失败，程序将继续运行但不支持远程控制功能。

## 程序工作流程

程序的工作流程如下：

1. **程序启动**：程序启动后，开始监控指定的进程
2. **状态检测**：每隔设定的检测间隔检查一次进程状态
3. **进程重启**：如果检测到进程不存在，等待重启延迟后重新启动
4. **次数统计**：记录重启次数，超过最大重启次数后停止监控
5. **用户交互**：用户可以随时输入 `exit` 指令退出程序
6. **远程控制**：如果启用了MQTT功能，将同时监控远程指令

## 重要注意事项

### 路径处理
1. **路径引用**：可执行文件路径如果包含空格，必须用引号括起来

### 权限要求
2. **权限要求**：确保有权限访问要重启的可执行文件

### 系统限制
3. **检测间隔**：检测间隔不能小于100毫秒

### 进程管理
4. **控制台复用**：被重启的程序会在新的控制台窗口中运行
5. **进程可见性**：被监控的程序必须在系统进程列表中可见

### 功能兼容性
6. **MQTT功能**：MQTT功能为可选功能，未编译时不会影响基础功能

## 最佳实践指南

### 服务器应用监控配置

```batch
console_auto_restart.exe server.exe "D:\Server\server.exe" -c 10000 -m 100
```

配置说明：
- 使用较长的检测间隔（10秒）减少系统开销
- 设置较大的最大重启次数（100次）

### 开发测试配置

```batch
console_auto_restart.exe testapp.exe "D:\Dev\testapp.exe" -l debug -d 1000
```

配置说明：
- 使用 DEBUG 日志等级查看详细信息
- 使用较短的重启延迟快速测试

### 生产环境配置（含MQTT）

```batch
console_auto_restart.exe app.exe "C:\App\app.exe" -c 5000 -d 3000 -m 20 -l info
```

配置说明：
- 使用合理的默认参数
- 使用 INFO 日志等级记录重要信息
- 启用MQTT功能以支持远程监控

## 常见问题及解决方案

### 问题：程序无法启动目标进程

**问题描述**：程序尝试启动目标进程时失败

**解决方案**：
- 检查可执行文件路径是否正确
- 确保有足够的权限
- 检查可执行文件是否损坏

### 问题：进程频繁重启

**问题描述**：目标进程在重启后立即退出，导致频繁重启

**解决方案**：
- 增加重启延迟时间
- 检查目标程序是否有严重bug
- 调整检测间隔

### 问题：无法输入指令

**问题描述**：在程序运行时无法输入控制指令

**解决方案**：
- 确保在程序的控制台窗口中输入
- 每个指令后按回车键
- 使用 `exit` 而不是 `q` 键退出

### 问题：MQTT连接失败

**问题描述**：程序启动时MQTT连接失败

**解决方案**：
- 确保MQTT服务器正在运行（默认localhost:1883）
- 检查网络连接
- 程序将继续运行但不支持远程控制功能
