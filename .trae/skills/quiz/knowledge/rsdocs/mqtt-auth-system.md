# MQTT 鉴权系统文档

## 概述

MQTT鉴权系统提供了一套完整的消息验证和权限控制机制，确保只有经过授权的设备能够执行特定操作。系统包含设备验证、消息完整性校验、权限分级管理等功能。

## 鉴权机制

### 1. 预共享密钥 (PSK)
- 使用预设的密钥对消息帧头进行加密
- 确保消息来源的可信性

### 2. 设备ID白名单
- 维护允许连接的设备ID列表
- 阻止未授权设备的访问

### 3. 数据完整性校验
- 使用CRC16算法验证消息完整性
- 防止消息在传输过程中被篡改

### 4. 权限分级系统
系统实现多级权限控制：

- **none**: 基本连接权限
- **basic**: 基础操作权限（状态查询等）
- **advanced**: 高级操作权限（备份等）
- **admin**: 管理员权限（重启、停止、CMD执行等）

## 帧结构

### 普通帧结构 (22字节最小)
- **帧头 (20字节)**:
  - [0-3]: 设备ID (4字节)
  - [4]: 帧类型 (1字节)
  - [5-6]: 数据长度 (2字节, 大端格式)
  - [7-10]: 时间戳 (4字节, 秒级)
  - [11-19]: 预留 (9字节)

- **数据段**: 实际负载数据
- **CRC校验 (2字节)**: 数据完整性验证

### CMD命令帧结构
- **帧类型**: 0x01 (CMD请求), 0x02 (CMD响应)
- **特殊字段**:
  - 命令ID (用于匹配请求和响应)
  - 权限级别验证

## 安全特性

### 1. 命令白名单
- 配置允许执行的命令列表
- 防止危险命令的执行

### 2. 危险命令拦截
自动拦截以下危险命令：
- 文件操作: `del`, `rm`, `rmdir`, `format`
- 系统控制: `shutdown`, `reboot`, `poweroff`, `halt`
- 进程管理: `kill`, `taskkill`, `tasklist`
- 用户管理: `net user`, `net localgroup`
- 系统工具: `reg`, `sc`, `schtasks`, `at`, `attrib`, `cipher`

### 3. 输出限制
- 限制命令输出长度 (默认8KB)
- 防止大输出导致的网络拥塞

## 配置说明

### 主要配置项

```json
{
  "modules": {
    "mqtt": {
      "commands": {
        "cmd": {
          "enabled": false,           // 是否启用CMD执行功能
          "permission": "admin",      // 执行权限级别
          "allowedCommands": ["dir", "echo", "ping"],  // 允许的命令
          "allowedPrefixes": ["dir ", "echo ", "ping "]  // 允许的命令前缀
        }
      }
    }
  },
  "businessLogic": {
    "enableCmdExecution": false,       // 业务逻辑层CMD执行开关
    "allowedCmds": ["dir", "echo"],    // 允许的命令列表
    "allowedCmdPrefixes": ["dir ", "echo "]  // 允许的命令前缀
  }
}
```

## 使用方式

### 1. 普通命令执行
发布消息到主题: `cmd/exec/{processName}`
消息内容: 要执行的命令

### 2. 加密CMD命令执行
使用加密帧格式发送CMD命令，系统会自动解析并验证。

## 最佳实践

1. **安全原则**: 默认禁用CMD执行功能
2. **权限控制**: 严格限制能够执行CMD的设备权限
3. **命令限制**: 仅允许必要且安全的命令
4. **监控审计**: 记录所有CMD执行日志
5. **网络隔离**: 在安全的网络环境中使用

## 故障排除

### 常见问题
- **命令被拒绝**: 检查命令是否在允许列表中
- **权限不足**: 确认设备权限级别配置正确
- **鉴权失败**: 验证PSK和设备ID配置

### 日志分析
系统会记录详细的鉴权和执行日志，用于故障排查和安全审计。
