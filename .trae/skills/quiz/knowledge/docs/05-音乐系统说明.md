# 音乐系统功能说明

## 概述

音乐系统是一个完整的音乐点播和收藏功能模块，支持多个音乐平台的点歌、选歌、收藏等功能。

## 支持的音源

| 音源 | API提供商 | 特点 |
|------|-----------|------|
| 汽水 | 笒鬼鬼API | 默认音源，返回标准JSON格式 |
| 酷我 | OIAPI | 返回纯文本格式，特殊处理 |
| 网易云 | OIAPI | 支持点歌 |
| QQ | 云汐API | 支持点歌 |

## 指令列表

### 菜单类

#### 音乐功能/音乐系统/音乐菜单
- **权限**: USER（需要卡密授权）
- **描述**: 显示音乐系统使用说明

### 点歌类

#### 点歌
- **权限**: USER
- **用法**: `点歌 [歌曲名]` 或 `[音源]点歌 [歌曲名]`
- **示例**: 
  - `点歌 小团圆`（使用默认音源）
  - `QQ点歌 小团圆`（使用QQ音乐）
  - `汽水点歌 小团圆`（使用汽水音乐）
  - `酷我点歌 小团圆`（使用酷我音乐）
  - `网易云点歌 小团圆`（使用网易云音乐）

#### QQ点歌/汽水点歌/酷我点歌/网易云点歌
- **权限**: USER
- **用法**: `[音源]点歌 [歌曲名]`
- **描述**: 指定音源搜索歌曲

### 选歌类

#### 选歌
- **权限**: USER
- **用法**: `选歌 [序号]` 或 `[方式]选歌 [序号]`
- **方式**: 卡片、语音、链接
- **示例**: 
  - `选歌 1`（默认卡片方式）
  - `卡片选歌 1`
  - `语音选歌 1`
  - `链接选歌 1`

### 收藏管理类

#### 个人歌单/我的收藏
- **权限**: USER
- **描述**: 查看个人收藏的歌单

#### 收藏歌曲/收藏音乐
- **权限**: USER
- **用法**: `收藏歌曲 [序号]`
- **描述**: 将搜索结果中的歌曲收藏到歌单

#### 取消收藏
- **权限**: USER
- **用法**: 
  - `取消收藏 [序号]`（取消单首）
  - `取消收藏 [起始序号]-[结束序号]`（批量取消）
  - `取消收藏 [起始序号]_[结束序号]`
- **示例**: 
  - `取消收藏 1`
  - `取消收藏 2-5`
  - `取消收藏 3_7`

#### 播放收藏
- **权限**: USER
- **用法**: `播放收藏 [序号]` 或 `[方式]播放收藏 [序号]`
- **方式**: 卡片、语音、链接
- **示例**: 
  - `播放收藏 1`
  - `卡片播放收藏 1`
  - `语音播放收藏 1`
  - `链接播放收藏 1`

#### 清空个人收藏歌曲/清空个人收藏音乐
- **权限**: USER
- **描述**: 清空个人收藏歌单

## 输出方式说明

### 卡片方式
- **描述**: 使用QQ音乐卡片展示歌曲
- **内容**: 封面、歌名、歌手、音频链接
- **优点**: 界面美观，可直接播放

### 链接方式
- **描述**: 以文本方式展示歌曲信息
- **内容**: 封面图片、歌名、歌手、音频链接
- **优点**: 可以复制链接，便于分享

### 语音方式
- **描述**: 直接发送语音消息
- **内容**: 音频链接
- **优点**: 自动播放，适合听歌

## API接口

### 音源API配置

```typescript
const MUSIC_API_URLS = {
  'QQ': (name, n) => `https://a.aa.cab/qq.music?msg=${name}&num=10&n=${n}`,
  '汽水': (name, n) => `https://api-v2.cenguigui.cn/api/qishui/?msg=${name}&type=json&n=${n}`,
  '酷我': (name, n) => `https://oiapi.net/api/Kuwo?msg=${name}&n=${n}&limit=10`,
  '网易云': (name, n) => `https://oiapi.net/api/Music_163?name=${name}&limit=20&n=${n}`
};
```

### 数据解析

不同音源返回的数据结构不同，系统会自动解析：

#### 汽水音乐（标准格式）
```json
{
  "code": 200,
  "data": [
    {
      "title": "小团圆",
      "singer": "陈雪燃",
      "cover": "封面URL",
      "music": "音频URL"
    }
  ]
}
```

#### 酷我音乐（非标准格式）
```json
{
  "code": 1,
  "message": "1. 你好    ——    初音ミク\n2. 你好    ——    Richard Clayderman\n..."
}
```
- **说明**: 酷我API返回纯文本格式，直接显示 `message` 内容
- **状态码**: `code: 1` 表示成功

#### QQ音乐
```json
{
  "song": "小团圆",
  "singer": "陈雪燃",
  "cover": "封面URL",
  "music": "音频URL"
}
```

#### 网易云音乐
```json
{
  "name": "小团圆",
  "singers": [{"name": "陈雪燃"}],
  "picurl": "封面URL",
  "url": "音频URL"
}
```

## 状态码处理

### API响应状态码

不同API使用不同的状态码格式：

| API | 成功状态码 | 说明 |
|-----|-----------|------|
| 汽水 | 200 | HTTP标准状态码 |
| 酷我 | 1 | 自定义状态码，数据在message字段 |
| 其他 | 0 | go-cqhttp标准状态码 |

### 状态码检查逻辑

```typescript
// 支持多种状态码格式
if (API数据.code !== 0 && API数据.code !== 1 && API数据.code !== 200) {
  await sendReplyMessage(ctx, event, `【${音源}】接口错误: ${API数据.message || '未知错误'}`);
  return;
}

// 特殊处理酷我API
if (音源 === '酷我' && !API数据.data && API数据.message) {
  await sendReplyMessage(ctx, event, API数据.message);
  return;
}
```

#### 2. 点歌处理

```typescript
async function handleSearchMusic(
  message: string,
  event: OB11Message,
  ctx: NapCatPluginContext,
  parsed: Record<string, string>
): Promise<void> {
  const [, 音源前缀, dd_name] = match;
  
  // 验证歌曲名称
  if (!dd_name || dd_name.trim() === '') {
    await sendReplyMessage(ctx, event, '你是在点歌嘛？');
    return;
  }
  
  // 确定音源
  const 音源 = 音源前缀 || getUserMusicSource(userId);
  if (音源前缀) {
    setUserMusicSource(userId, 音源前缀);
  }
  
  // 调用API
  const apiUrl = MUSIC_API_URLS[音源](dd_name);
  const API数据 = await fetchMusicData(apiUrl);
  
  // 检查响应状态码（支持不同API的状态码格式：0、1或200都表示成功）
  if (API数据.code !== 0 && API数据.code !== 1 && API数据.code !== 200) {
    await sendReplyMessage(ctx, event, `【${音源}】接口错误: ${API数据.message || '未知错误'}`);
    return;
  }
  
  // 特殊处理酷我API的非标准格式
  if (音源 === '酷我' && !API数据.data && API数据.message) {
    await sendReplyMessage(ctx, event, API数据.message);
    return;
  }
  
  // 解析歌单并保存临时数据
  const 歌曲列表 = parseMusicList(音源, API数据.data);
  saveTempPlaylist(userId, { 音源, data: 歌曲列表 });
  
  // 返回搜索结果
  await sendReplyMessage(ctx, event, 组装消息);
}
```

#### 3. 选歌处理

```typescript
async function handleSelectMusic(
  message: string,
  event: OB11Message,
  ctx: NapCatPluginContext,
  parsed: Record<string, string>
): Promise<void> {
  const [, 方式前缀, mub] = match;
  const userId = String(event.user_id);
  
  // 获取临时数据
  const 最大范围 = pluginState.loadDataFile('筱筱吖/音乐系统/选歌范围.json', userId, 0);
  const dd_name = pluginState.loadDataFile('筱筱吖/音乐系统/点歌名字.json', userId, '');
  const 音源 = getUserMusicSource(userId);
  
  // 验证序号
  const 序号 = Number(mub);
  if (序号 === 0 || 序号 > 最大范围) {
    await sendReplyMessage(ctx, event, `可选范围异常，当前你可选范围为【${最大范围}】`);
    return;
  }
  
  // 调用API获取歌曲详情
  const apiUrl = MUSIC_API_URLS[音源](dd_name, 序号);
  const API数据 = await fetchMusicData(apiUrl);
  
  // 特殊处理酷我API
  if (音源 === '酷我' && !API数据.data && API数据.message) {
    await sendReplyMessage(ctx, event, API数据.message);
    return;
  }
  
  // 解析歌曲信息
  const 歌曲信息 = parseMusicData(音源, API数据.data);
  
  // 根据方式输出
  const 方式 = 方式前缀 || '卡片';
  if (方式 === '卡片') {
    const musicCard = `[CQ:music,type=custom,...]`;
    await sendReplyMessage(ctx, event, musicCard);
  } else if (方式 === '链接') {
    await sendReplyMessage(ctx, event, 组装输出);
  } else if (方式 === '语音') {
    await sendReplyMessage(ctx, event, `[CQ:record,url=${歌曲信息.music}]`);
  }
}
```

    await sendReplyMessage(ctx, event, `已删除${序号1}|${序号2 || 序号1}\n...`);
  }

## 技术要点

### 1. 多API状态码兼容

系统支持不同API的状态码格式：

```typescript
// 支持三种成功状态码
if (API数据.code !== 0 && API数据.code !== 1 && API数据.code !== 200) {
  // 错误处理
}
```

### 2. 非标准API格式处理

酷我API返回纯文本格式，需要特殊处理：

```typescript
// 酷我API：数据在message字段中
if (音源 === '酷我' && !API数据.data && API数据.message) {
  await sendReplyMessage(ctx, event, API数据.message);
  return;
}
```

### 3. 减少if嵌套

通过提前return和函数提取减少嵌套：

```typescript
// 重构前：多层嵌套
if (操作 === '收藏歌曲') {
  if (临时歌单) {
    if (序号1 > 0) {
      // 处理逻辑
    }
  }
}

// 重构后：提前return
if (操作 === '收藏歌曲' || 操作 === '收藏音乐') {
  const 临时歌单 = getTempPlaylist(userId);
  if (!临时歌单) {
    await sendReplyMessage(ctx, event, '请先点歌再操作！');
    return;
  }
  
  if (序号1 === 0 || 序号1 > 临时歌单.data.length) {
    await sendReplyMessage(ctx, event, '选取范围无效哦！');
    return;
  }
  
  // 处理逻辑
}
```

### 4. 数据一致性

- 所有数据操作使用 pluginState 统一接口
- 使用 JSON 格式存储复杂数据
- 临时数据和持久化数据分离

### 5. 错误处理

- API请求失败时返回友好提示
- 数据解析失败时提供默认值
- 参数验证失败时提前返回

### 6. 音源适配

不同音源的数据结构不同，系统通过 `parseMusicData` 和 `parseMusicList` 函数自动适配：

```typescript
function parseMusicData(source: string, data: any): MusicSourceData {
  switch (source) {
    case '汽水':
      return {
        title: data.title,
        singer: data.singer,
        cover: data.cover,
        music: data.music
      };
    // ... 其他音源
  }
}
```

## 注意事项

1. **授权要求**: 音乐系统功能需要卡密授权
2. **API依赖**: 依赖外部API接口，接口失效时无法使用
3. **临时数据**: 点歌后需要立即选歌，否则临时数据会丢失
4. **收藏限制**: 暂无收藏数量限制
5. **输出方式**: 不同输出方式适用于不同场景
6. **酷我特殊格式**: 酷我API返回纯文本，无法使用卡片/语音/链接格式
