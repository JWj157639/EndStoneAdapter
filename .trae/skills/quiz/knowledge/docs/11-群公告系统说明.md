# 群公告系统说明

## 功能概述

群公告系统提供了便捷的群公告发布功能，支持：
- 纯文本公告发布
- 图文公告发布
- 自动置顶显示
- 智能图片处理

## 模块文件

```
src/modules/
├── announcement.commands.ts  # 群公告指令定义和处理逻辑
└── announcement.module.ts    # 群公告系统菜单
```

## 使用方法

### 1. 命令列表

| 命令 | 权限 | 说明 |
|------|------|------|
| `群公告系统` | 公开 | 显示群公告系统菜单 |
| `设置群公告 [内容] [可选图片]` | 群主/管理员 | 发布群公告 |

### 2. 使用示例

#### 发送纯文本公告
```
设置群公告 本周六晚上8点召开全员会议，请准时参加
```

#### 发送图文公告
```
设置群公告 重要通知：服务器维护时间 [附带图片]
```

#### 查看帮助菜单
```
群公告系统
```

## 技术实现

### 1. 图片处理流程

当用户发送带图片的群公告时，系统会按以下流程处理：

```
用户发送消息
    ↓
提取图片信息（file 字段）
    ↓
调用 get_image API 获取本地路径
    ↓
使用本地路径调用 _send_group_notice API
    ↓
发布成功
```

### 2. 关键代码逻辑

#### 提取公告文本
```typescript
function extractAnnouncementText(rawMessage: string): string | null {
  // 检查命令前缀：设置群公告
  if (!rawMessage.startsWith('设置群公告 ')) {
    return null;
  }
  
  // 去掉前缀，获取公告内容
  const content = rawMessage.substring('设置群公告 '.length).trim();
  
  // 去除 CQ 码图片部分
  return content.replace(/\[CQ:image[^\]]*\]/g, '').trim();
}
```

#### 处理图片
```typescript
async function processImage(
  ctx: NapCatPluginContext,
  imageFile: string
): Promise<string | null> {
  try {
    const result = await ctx.actions.call('get_image', {
      file: imageFile
    }, ctx.adapterName, ctx.pluginManager.config);
    
    // 返回本地文件路径
    return result?.file || null;
  } catch (error) {
    ctx.logger.error('[processImage] 处理图片失败:', error);
    return null;
  }
}
```

#### 构建公告参数
```typescript
function buildNoticeParams(groupId: string, content: string, imagePath?: string): Record<string, any> {
  return {
    group_id: groupId,
    content,
    pinned: 1,              // 置顶
    type: 1,                // 公告类型
    confirm_required: 0,    // 不需要确认
    is_show_edit_card: 1,   // 显示编辑卡片
    tip_window_type: 0,     // 提示窗口类型
    ...(imagePath && { image: imagePath })  // 可选图片
  };
}
```

## API 调用说明

### _send_group_notice API

**请求参数：**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| group_id | string/number | 是 | 群号ID |
| content | string | 是 | 公告文本内容 |
| image | string | 否 | 图片路径（本地路径/URL/base64） |
| pinned | number/string | 是 | 是否置顶（0/1） |
| type | number/string | 是 | 公告类型 |
| confirm_required | number/string | 是 | 是否需要成员确认（0/1） |
| is_show_edit_card | number/string | 是 | 是否显示编辑卡片（0/1） |
| tip_window_type | number/string | 是 | 提示窗口类型 |

**图片参数支持格式：**

1. **本地文件路径**（推荐）：
   ```
   /root/.config/QQ/nt_qq_xxx/nt_data/Pic/2026-03/Ori/image.jpg
   ```

2. **公网可访问的图片URL**：
   ```
   https://example.com/image.png
   ```

3. **base64 格式**：
   ```
   base64://image/jpeg,/9j/4AAQSkZJRg...
   ```

⚠️ **重要提示：**
- 不支持腾讯云临时URL（会导致 "pic format is incorrect" 错误）
- 如果使用QQ消息中的图片，建议先调用 `get_image` API 获取本地路径

## 常见问题

### 1. 权限不足

**错误信息：**
```
窝没有权限设置群公告唉～
```

**解决方案：**
- 确保机器人账号在群中具有管理员或群主权限
- 检查命令是否由群主或管理员发送

### 2. 图片格式错误

**错误信息：**
```
图片处理失败，请重试或只发送文本公告
```

**可能原因：**
- 使用了腾讯云临时URL
- 图片文件损坏
- `get_image` API 返回失败

**解决方案：**
- 重新发送图片
- 只发送文本公告

### 3. API 返回 "No data returned"

**现象：**
群公告实际已成功发送，但 API 返回 "No data returned"

**说明：**
这是 NapCat 的正常行为，虽然返回 "No data returned"，但群公告实际已成功发送。

**处理方式：**
系统已自动处理此情况，会返回成功消息给用户。

## 开发注意事项

### 1. 图片处理

- 必须使用 `get_image` API 获取本地路径，不能直接使用 URL
- 腾讯云临时 URL 不支持，会导致格式错误
- 处理图片时需要添加错误处理，失败时降级为纯文本公告

### 2. 错误处理

- 统一使用 `parseErrorMessage` 函数解析错误
- 特殊处理 "No data returned" 情况
- 所有错误都应返回用户友好的提示

### 3. 日志记录

- 使用 `ctx.logger.debug` 记录调试信息
- 使用 `ctx.logger.error` 记录错误信息
- 日志格式应统一，便于排查问题

### 4. 代码规范

- 使用常量避免魔法值
- 拆分复杂逻辑为独立函数
- 使用 TypeScript 类型定义确保类型安全
- 使用可选链和空值合并简化代码
