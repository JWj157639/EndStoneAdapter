# QTBOT 插件 - 项目架构文档

## 概述

QTBOT 是一个基于 NapCat 框架的 QQ 机器人插件，采用模块化架构设计，实现了完整的三层授权 + 指令权限系统。

## 技术栈

- **语言**: TypeScript
- **框架**: NapCat Plugin
- **构建工具**: Vite
- **依赖管理**: pnpm
- **前端**: React + TailwindCSS (WebUI)

## 核心组件

### 1. 指令注册系统

**文件**: `src/core/command-registry.ts`

- **职责**: 管理所有指令的注册、查询和匹配
- **功能**:
  - 指令元数据存储
  - 精确匹配、前缀匹配、正则匹配
  - 指令查询和统计

### 2. 权限校验系统

**文件**: `src/core/permission-checker.ts`

- **职责**: 三层权限校验
- **功能**:
  - 白名单校验
  - 卡密授权校验
  - 角色权限校验
  - 核心API: `checkUserPermission()`

### 3. 用户角色管理

**文件**: `src/core/user-role-manager.ts`

- **职责**: 识别和管理用户角色
- **角色类型**:
  - OWNER: 主人（所有权限）
  - USER: 普通用户（已授权）
  - GUEST: 游客（未授权）

### 4. 全局状态管理

**文件**: `src/core/state.ts`

- **职责**: 单例模式管理全局状态
- **功能**:
  - 配置加载和保存
  - 数据文件读写
  - 统计信息管理
  - 定时器管理

### 5. 消息处理器

**文件**: `src/handlers/message-handler.ts`

- **职责**: 处理所有接收到的消息
- **流程**:
  1. 检查插件启用状态
  2. 检查群组启用状态
  3. 检查冷却时间
  4. 解析消息内容
  5. 匹配指令
  6. 三层权限校验
  7. 执行指令处理器

## 三层权限系统详解

### 第一层：白名单校验（全局主授权）

- **检查项**:
  - 全局开关是否启用
  - 群聊/好友白名单检查
- **规则**:
  - 白名单为空时，拒绝所有
  - 主人指令可设置 `requireWhitelist: false` 跳过白名单

### 第二层：卡密授权校验

- **检查项**:
  - 用户是否为主人（主人无需卡密）
  - 卡密有效性检查
  - 卡密过期检查
- **授权类型**:
  - OWNER: 主人授权（无需卡密）
  - CARD: 卡密授权
  - NONE: 未授权

### 第三层：角色权限校验

- **权限等级**:
  - PUBLIC (0): 公开指令，所有用户
  - USER (1): 用户指令，需要卡密授权
  - OWNER (2): 主人指令，仅主人
- **规则**:
  - 用户角色等级 >= 指令权限等级 → 允许
  - 用户角色等级 < 指令权限等级 → 拒绝

## 模块化设计

### 指令注册流程

```
系统启动 (plugin_init)
    ↓
导入所有模块指令定义
    ↓
批量注册到 CommandRegistry
    ↓
指令注册完成
```

### 消息处理流程

```
接收消息
    ↓
解析消息内容
    ↓
匹配指令
    ↓
三层权限校验
    ├─ 第一层：白名单校验
    ├─ 第二层：卡密授权校验
    └─ 第三层：角色权限校验
    ↓
执行指令处理器
    ↓
返回结果
```

## 数据存储

### 配置文件

- **位置**: `config.json` (NapCat 配置目录)
- **内容**: 插件配置、主人QQ、白名单等

### 数据文件

- **卡密数据**: `筱筱吖/授权系统/卡密管理/卡密数据.json`
- **授权信息**: `筱筱吖/授权系统/授权信息/{群号|私聊}.json`
- **签到数据**: `筱筱吖/娱乐系统/签到数据/`
- **游戏数据**: `筱筱吖/娱乐系统/游戏数据/`

## 扩展性设计

### 添加新模块

1. 创建模块文件 `src/modules/{module}.module.ts`
2. 定义指令元数据数组
3. 在 `src/modules/index.ts` 中导入并导出
4. 系统启动时自动注册

### 添加新指令

```typescript
export const newCommand: CommandMetadata = {
  name: '新指令',
  pattern: /^新指令$/,
  level: CommandLevel.USER,
  module: '模块名',
  description: '指令描述',
  handler: async (message, event, ctx, parsed) => {
    // 业务逻辑
  },
  requireAuth: true
};
```

## 安全性设计

1. **权限隔离**: 三层权限校验，防止权限提升
2. **白名单强制**: 除主人指令外，必须通过白名单
3. **卡密唯一使用**: 卡密使用后立即删除
4. **操作审计**: 所有关键操作记录日志
5. **冷却时间限制**: 防止暴力破解

## 性能优化

1. **指令缓存**: 指令注册表缓存在内存中
2. **快速匹配**: 支持多种匹配模式，优先精确匹配
3. **异步处理**: 消息处理采用异步模式
4. **状态管理**: 单例模式避免重复初始化

## 生命周期管理

### plugin_init (插件初始化)

- 初始化全局状态
- 生成配置 Schema
- 注册 WebUI 页面
- 注册 API 路由
- 注册所有指令

### plugin_onmessage (消息处理)

- 接收消息事件
- 解析和分发消息
- 权限校验
- 执行指令

### plugin_onevent (事件处理)

- 处理通知事件
- 处理请求事件
- 处理拍一拍等特殊事件

### plugin_cleanup (插件清理)

- 清理已注册的指令
- 清理定时器
- 保存配置
- 释放资源
