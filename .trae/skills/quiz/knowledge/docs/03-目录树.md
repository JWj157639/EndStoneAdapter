## 核心目录说明

### src/ - 源代码目录

项目的主要源代码，包含所有核心功能实现。

#### config.ts
**作用**: 插件配置定义
- 默认配置常量
- WebUI 配置 Schema 构建
- 配置验证

#### index.ts
**作用**: 插件主入口
- plugin_init: 插件初始化
- plugin_onmessage: 消息处理
- plugin_onevent: 事件处理
- plugin_cleanup: 插件清理
- 配置管理钩子

#### types.ts
**作用**: 类型定义
- 插件配置类型
- 群配置类型
- 好友配置类型
- 消息处理相关类型

---

### src/core/ - 核心模块

包含插件的核心功能，特别是新的权限系统。

#### auth-system.ts
**作用**: 旧版授权系统（保留兼容）
- 用户权限检查
- 主人权限检查
- 卡密生成和验证
- 卡密使用和记录

#### command-registry.ts 【新增】
**作用**: 指令注册表
- 指令注册（单个/批量）
- 指令匹配（精确/前缀/正则）
- 指令查询（名称/模块/等级）
- 指令统计和清理

#### command-matcher.ts 【新增】
**作用**: 指令匹配器
- 消息文本提取
- 命令前缀处理
- 消息指令解析

#### permission-checker.ts 【新增】
**作用**: 权限校验器（核心API）
- 第一层：白名单校验
- 第二层：卡密授权校验
- 第三层：角色权限校验
- 核心函数：checkUserPermission()
- 批量权限检查
- 可用指令查询

#### permission-types.ts 【新增】
**作用**: 权限系统类型定义
- UserRole 枚举（角色类型）
- CommandLevel 枚举（权限等级）
- AuthType 枚举（授权类型）
- CommandMetadata 接口（指令元数据）
- PermissionCheckResult 接口（权限检查结果）

#### scheduler.ts
**作用**: 定时任务调度器
- 定时任务管理
- 任务执行
- 任务清理

#### state.ts
**作用**: 全局状态管理（单例模式）
- 插件上下文管理
- 配置加载和保存
- 数据文件读写
- 统计信息管理
- 定时器管理

#### user-role-manager.ts 【新增】
**作用**: 用户角色管理器
- 角色识别（OWNER/USER/GUEST）
- 主人检查
- 卡密授权检查
- 授权过期时间查询
- 角色等级转换

## 权限系统架构

### 三层权限校验

```
用户请求
    ↓
【第一层】白名单校验
    ├─ 全局开关检查
    └─ 白名单验证
    ↓
【第二层】卡密授权校验
    ├─ 主人检查
    └─ 卡密有效性检查
    ↓
【第三层】角色权限校验
    ├─ 用户角色识别
    └─ 权限等级匹配
    ↓
允许/拒绝
```

### 角色与权限等级

| 角色 | 等级 | 描述 |
|------|------|------|
| OWNER | 2 | 主人，所有权限 |
| USER | 1 | 普通用户，已授权 |
| GUEST | 0 | 游客，未授权 |

| 指令等级 | 数值 | 描述 |
|---------|------|------|
| PUBLIC | 0 | 公开指令，所有用户 |
| USER | 1 | 用户指令，需要授权 |
| OWNER | 2 | 主人指令，仅主人 |
