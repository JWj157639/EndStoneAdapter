# 签到系统功能说明

## 概述

签到系统是日常功能模块的核心功能，用户每日签到可获得归笺奖励，系统会记录用户的累计签到天数和连续签到天数。

## 指令

### 签到
- **指令名称**: `签到`
- **别名**: `打卡`
- **权限等级**: USER（需要卡密授权）
- **描述**: 每日签到获取归笺奖励

## 功能特性

### 1. 奖励机制

签到奖励基于用户当天的签到排名，排名越早，奖励越丰厚：

| 排名 | 奖励范围（归笺） |
|------|-----------------|
| 第1名 | 90-125 |
| 第2名 | 75-89 |
| 第3名 | 50-74 |
| 其他 | 15-49 |

### 2. 连续签到计算

系统会自动计算用户的连续签到天数：

- **正常连续签到**: 两次签到间隔在36小时（129600秒）以内
- **连签中断**: 两次签到间隔超过36小时，连签天数重置为1
- **首次签到**: 连签天数记录为1

### 3. 数据记录

系统会记录以下签到相关数据：

#### 个人数据
- **累计签到天数**: 用户总签到次数
- **连续签到天数**: 当前连续签到天数
- **上次签到时间**: 最后一次签到的时间戳

#### 每日数据
- **全服签到人数**: 当天全服签到总人数
- **签到状态**: 用户当天是否已签到
- **签到排名**: 用户当天的签到排名
- **签到详细时间**: 用户签到时的具体时间

## 技术要点

### 1. 减少if嵌套

原始代码中有多层if嵌套，重构后通过提前return减少嵌套层级：

```typescript
// 重构前：多层嵌套
if (签到状态 !== '未知') {
  // ...
  return;
}
// 继续处理...

// 重构后：提前return
if (签到状态 !== '未知') {
  let 返回内容 = '❌你今天签到过啦～...';
  await sendReplyMessage(ctx, event, 返回内容);
  return;
}
// 继续处理...
```

### 2. 数据一致性

- 所有数据操作使用 pluginState 的统一接口
- 避免直接操作文件系统
- 确保数据写入的原子性

### 3. 时间处理

- 使用时间戳（秒）进行时间差计算
- 36小时（129600秒）作为连签中断的临界值
- 使用 formatTimestamp 工具函数格式化时间显示

### 4. 随机数生成

使用工具函数 `rand(min, max)` 生成指定范围内的随机整数：

```typescript
import { rand } from '../utils/money-utils';

const 奖励 = rand(90, 125); // 生成 90-125 之间的随机数
```

## 注意事项

1. **每日限制**: 每个用户每天只能签到一次
2. **授权要求**: 签到功能需要卡密授权
3. **数据持久化**: 所有签到数据持久化存储，重启不丢失
4. **并发处理**: 系统会按签到顺序自动分配排名
5. **时间校准**: 签到排名基于实际签到时间，不是消息接收时间
