# 网络线程阻塞问题 - 解决方案

## 问题概述

插件运行约 2 小时后出现网络线程阻塞现象，影响系统正常运行。

## 根本原因分析

### 1. 文件 I/O 阻塞 ⚠️ 高风险
- **位置**: `src/core/state.ts` 中的 `loadDataFile` 和 `saveDataFile` 方法
- **问题**: 使用 `fs.readFileSync` 和 `fs.writeFileSync` 同步读写文件
- **影响**: 每次消息处理都进行多次同步文件操作，长时间运行后累积阻塞效应明显
- **优先级**: 🔴 最高

### 2. HTTP 连接池资源泄漏 ⚠️ 高风险
- **位置**: `src/utils/http-client.ts`
- **问题**: 
  - `maxSockets: 50` 连接池大小不足
  - 没有连接池清理机制
  - 熔断器状态可能永远不会重置
- **影响**: 长时间运行后连接池被耗尽，导致无法发起新的 HTTP 请求
- **优先级**: 🔴 最高

### 3. 定时任务串行执行 ⚠️ 中风险
- **位置**: `src/services/cron-service.ts`
- **问题**: 整点报时和全群打卡任务串行遍历所有群发送消息
- **影响**: 群数量多时会阻塞较长时间
- **优先级**: 🟡 中等

### 4. 事件处理器串行处理 ⚠️ 中风险
- **位置**: `src/handlers/event-handler.ts`
- **问题**: 所有通知和请求事件串行处理，没有并发限制
- **影响**: 高频事件时可能累积阻塞
- **优先级**: 🟡 中等

### 5. 内存泄漏风险 ⚠️ 低风险
- **位置**: `src/core/error-handler.ts`
- **问题**: 错误记录数组没有自动清理机制
- **影响**: 长时间运行后会占用大量内存
- **优先级**: 🟢 低

## 解决方案（按优先级排序）

### 阶段一：紧急修复（最高优先级）

#### 1.1 文件 I/O 异步化

**目标**: 将所有同步文件操作改为异步操作，消除阻塞

**实施方案**:
```typescript
// src/core/state.ts

import { promises as fsp } from 'fs';

// 异步读取文件
async loadDataFileAsync<T>(filename: string, key: string, defaultValue: T): Promise<T> {
  const filePath = this.getDataFilePath(filename);
  try {
    const content = await fsp.readFile(filePath, 'utf-8');
    const data = JSON.parse(content);
    const value = data[key];
    return (value !== null && value !== undefined) ? value : defaultValue;
  } catch (e) {
    this.logger.warn(`读取数据文件 ${filename} 失败:`, e);
    return defaultValue;
  }
}

// 异步写入文件
async saveDataFileAsync<T>(filename: string, key: string, value: T): Promise<void> {
  const filePath = this.getDataFilePath(filename);
  const dir = path.dirname(filePath);
  
  try {
    await fsp.mkdir(dir, { recursive: true });
    
    let data: any = {};
    try {
      const content = await fsp.readFile(filePath, 'utf-8');
      data = JSON.parse(content);
    } catch (e) {
      // 文件不存在或解析失败，使用空对象
    }
    
    data[key] = value;
    await fsp.writeFile(filePath, JSON.stringify(data, null, 2));
  } catch (e) {
    this.logger.error(`保存数据文件 ${filename} 失败:`, e);
    throw e;
  }
}

// 保持向后兼容的同步方法（标记为已废弃）
/** @deprecated 使用 loadDataFileAsync 替代 */
loadDataFile<T>(filename: string, key: string, defaultValue: T): T {
  // 现有同步实现...
}

/** @deprecated 使用 saveDataFileAsync 替代 */
saveDataFile<T>(filename: string, key: string, value: T): void {
  // 现有同步实现...
}
```

**影响范围**: 
- `src/core/auth-system.ts` - 授权系统
- `src/core/permission-checker.ts` - 权限检查
- `src/core/event-manager.ts` - 事件管理
- `src/core/user-role-manager.ts` - 用户角色管理
- `src/services/scheduler.ts` - 定时调度

**性能提升**: 根据 benchmark，异步文件写入可提升 96.34% 性能

#### 1.2 HTTP 连接池优化

**目标**: 增加连接池大小，添加清理机制，防止资源泄漏

**实施方案**:
```typescript
// src/utils/http-client.ts

export class HttpClient {
  private agent: https.Agent;
  private cleanupInterval: NodeJS.Timeout;
  private resetInterval: NodeJS.Timeout;

  constructor(private ctx: NapCatPluginContext, private options: RetryOptions & CircuitBreakerOptions = {}) {
    const {
      threshold = 5,
      cooldown = 30000,
    } = options;

    this.circuitBreaker = new CircuitBreaker(threshold, cooldown);
    
    // 增加连接池大小
    this.agent = new https.Agent({
      keepAlive: true,
      keepAliveMsecs: 30000,  // 30秒保持连接
      maxSockets: 200,        // 从 50 增加到 200
      maxFreeSockets: 50,     // 从 10 增加到 50
      timeout: this.options.timeout || 60000,
      scheduling: 'fifo',     // FIFO 调度策略
    });

    // 定期清理空闲连接（每5分钟）
    this.cleanupInterval = setInterval(() => {
      this.cleanupConnections();
    }, 5 * 60 * 1000);

    // 每小时自动重置熔断器
    this.resetInterval = setInterval(() => {
      this.resetCircuitBreakerIfNeeded();
    }, 60 * 60 * 1000);
  }

  // 清理空闲连接
  private cleanupConnections(): void {
    try {
      const sockets = (this.agent as any).sockets || {};
      const freeSockets = (this.agent as any).freeSockets || {};
      
      const totalConnections = Object.keys(sockets).length;
      const freeConnections = Object.keys(freeSockets).length;
      
      this.ctx.logger.debug('[HttpClient] 连接池状态:', {
        total: totalConnections,
        free: freeConnections,
        active: totalConnections - freeConnections
      });

      // 如果空闲连接过多，销毁并重建 Agent
      if (freeConnections > 100) {
        this.ctx.logger.warn(`[HttpClient] 空闲连接过多 (${freeConnections})，重置连接池`);
        this.agent.destroy();
        
        this.agent = new https.Agent({
          keepAlive: true,
          keepAliveMsecs: 30000,
          maxSockets: 200,
          maxFreeSockets: 50,
          timeout: this.options.timeout || 60000,
          scheduling: 'fifo',
        });
        
        this.ctx.logger.info('[HttpClient] 连接池已重置');
      }
    } catch (error) {
      this.ctx.logger.error('[HttpClient] 清理连接失败:', error);
    }
  }

  // 重置熔断器（如果需要）
  private resetCircuitBreakerIfNeeded(): void {
    try {
      const state = this.circuitBreaker.getState();
      
      if (state === 'open') {
        this.ctx.logger.info('[HttpClient] 熔断器状态为 open，尝试重置');
        this.circuitBreaker.reset();
        this.ctx.logger.info('[HttpClient] 熔断器已自动重置');
      }
    } catch (error) {
      this.ctx.logger.error('[HttpClient] 重置熔断器失败:', error);
    }
  }

  // 清理资源
  destroy(): void {
    if (this.cleanupInterval) {
      clearInterval(this.cleanupInterval);
    }
    if (this.resetInterval) {
      clearInterval(this.resetInterval);
    }
    if (this.agent) {
      this.agent.destroy();
    }
    this.ctx.logger.info('[HttpClient] 资源已清理');
  }

  // 获取连接池状态
  getConnectionPoolStatus(): { total: number; free: number; active: number } {
    const sockets = (this.agent as any).sockets || {};
    const freeSockets = (this.agent as any).freeSockets || {};
    
    const total = Object.keys(sockets).length;
    const free = Object.keys(freeSockets).length;
    
    return {
      total,
      free,
      active: total - free
    };
  }
}

// 更新单例创建函数，添加 destroy 方法
let httpClientInstance: HttpClient | null = null;

export function getHttpClient(ctx: NapCatPluginContext): HttpClient {
  if (!httpClientInstance) {
    httpClientInstance = new HttpClient(ctx, {
      maxRetries: 3,
      baseDelay: 500,
      maxDelay: 60000,
      timeout: 60000,
      threshold: 5,
      cooldown: 30000,
    });
  }
  return httpClientInstance;
}

export function destroyHttpClient(): void {
  if (httpClientInstance) {
    httpClientInstance.destroy();
    httpClientInstance = null;
  }
}
```

**性能提升**: 启用 keepAlive 后，第二次请求快 70%

#### 2.2 事件处理并发控制

**目标**: 添加事件处理并发限制，防止高频事件累积阻塞

**实施方案**:
```typescript
// src/handlers/event-handler.ts

import pLimit from 'p-limit';

export class EventHandler {
  private eventLimit = pLimit(5); // 最多5个并发事件处理

  async handleNotice(event: any): Promise<void> {
    return this.eventLimit(async () => {
      try {
        this.ctx.logger.debug(`[EventHandler] 收到通知事件: ${event.notice_type}`);

        switch (event.notice_type) {
          case 'group_increase':
            await this.handleGroupIncrease(event);
            break;
          case 'group_decrease':
            await this.handleGroupDecrease(event);
            break;
          // ... 其他事件类型
        }
      } catch (error) {
        this.ctx.logger.error('[EventHandler] 处理通知事件失败:', error);
      }
    });
  }

  async handleRequest(event: any): Promise<void> {
    return this.eventLimit(async () => {
      try {
        this.ctx.logger.debug(`[EventHandler] 收到请求事件: ${event.request_type}`);

        switch (event.request_type) {
          case 'group':
            await this.handleGroupRequest(event);
            break;
          case 'friend':
            await this.handleFriendRequest(event);
            break;
        }
      } catch (error) {
        this.ctx.logger.error('[EventHandler] 处理请求事件失败:', error);
      }
    });
  }
}
```

### 阶段三：资源管理（低优先级）

#### 3.1 内存清理机制

**目标**: 定期清理错误记录，防止内存泄漏

**实施方案**:
```typescript
// src/core/error-handler.ts

export class ErrorHandler {
  private cleanupInterval: NodeJS.Timeout;

  constructor(private ctx: NapCatPluginContext) {
    // 每小时清理一次错误记录
    this.cleanupInterval = setInterval(() => {
      this.cleanupOldErrors();
    }, 60 * 60 * 1000);
  }

  private cleanupOldErrors(): void {
    try {
      const originalCount = this.errors.length;
      
      // 只保留最近的50条错误记录
      if (this.errors.length > 50) {
        this.errors = this.errors.slice(0, 50);
        this.ctx.logger.info(`[ErrorHandler] 错误记录已清理: ${originalCount} -> ${this.errors.length}`);
      }
    } catch (error) {
      this.ctx.logger.error('[ErrorHandler] 清理错误记录失败:', error);
    }
  }

  destroy(): void {
    if (this.cleanupInterval) {
      clearInterval(this.cleanupInterval);
    }
    this.clearErrors();
    this.ctx.logger.info('[ErrorHandler] 资源已清理');
  }
}
```
