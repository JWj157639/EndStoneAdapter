## 性能优化方法论

### 问题诊断流程
1. **定位瓶颈**：通过日志和监控识别阻塞点
2. **根本原因分析**：多维度排查（文件I/O、连接池、内存、定时任务）
3. **分优先级修复**：高风险 → 中风险 → 低风险

### 常见优化手段
- **文件 I/O 异步化**：用 `fs/promises` 替代同步方法，性能提升 96%
- **HTTP 连接池优化**：增加连接池大小，添加清理机制，防止资源泄漏
- **定时任务批量处理**：用 `p-limit` 控制并发，避免串行阻塞
- **内存清理机制**：定期清理错误记录，防止内存泄漏

### 性能验证
- 压力测试运行 4 小时以上
- 监控文件 I/O、HTTP 连接池、内存使用
- 日志分析阻塞点

## 安全设计原则

### 优先级：安全 > 效率 > 功能

#### 文件上传安全
- ZIP 文件解析使用纯 Buffer 操作，不转换为字符串
- 二进制签名搜索，避免编码问题
- Token 校验，防范 XXC 攻击

#### MQTT 鉴权架构
- **先鉴权后解析**：服务端先完成身份鉴权，仅合法消息才进入解析
- **加密帧头**：AES-ECB-128 加密 20 字节固定帧头
- **数据段纯二进制明文**：无加密开销，计算机原生解析
- **CRC16 完整性校验**：防传输篡改

#### 权限隔离
- 三层权限校验，防止权限提升
- 白名单强制（除主人指令外）
- 卡密唯一使用，用后即删

## 经验知识库

### 文件处理
- ZIP 文件解析必须用纯 Buffer，`body.toString('binary')` 会损坏文件
- 二进制签名搜索用 `Buffer.from([0x50, 0x4B, 0x03, 0x04])`，不用 `indexOf('PK')`

### NapCat 插件
- 新插件默认禁用，必须 `setPluginStatus(pluginId, true)` 启用
- `loadDirectoryPlugin` 参数是目录名，不是完整路径
- API 返回 "No data returned" 可能实际已成功

### 群公告图片
- 不支持腾讯云临时 URL（缺少文件后缀）
- 正确方式：`get_image` API 获取本地路径 → 使用本地路径发送
- 支持：本地路径、公网 URL（标准格式）、base64 格式

### 性能优化
- 异步文件写入性能提升 96%
- HTTP Keep-Alive 第二次请求快 70%
- 定时任务并发控制避免串行阻塞
