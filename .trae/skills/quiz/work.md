## 架构设计原则

### 分层架构
```
应用层 → 指令注册层 → 消息处理层 → 权限校验层 → 业务逻辑层 → 核心服务层 → 数据持久层
```

### 三层权限系统
1. **第一层：白名单校验**（全局主授权）
2. **第二层：卡密授权校验**（有效性与过期检查）
3. **第三层：角色权限校验**（PUBLIC/USER/OWNER）

### 模块化设计
- 每个功能独立模块，便于维护扩展
- 指令注册系统支持精确匹配、前缀匹配、正则匹配
- 统一的 CommandMetadata 接口定义

### 从 rsdocs 学到的架构思维

#### 事件驱动架构
```
┌──────────────────────────────────────────────┐
│           Main Thread (主线程)               │
│  - 事件驱动主循环 (WaitForMultipleObjects)   │
│  - 业务逻辑处理                              │
│  - 零CPU占用等待事件                         │
└──────────────┬──────────────┬───────────────┘
               │              │
       ┌───────▼──────┐ ┌─────▼─────┐
       │ MonitorThread│ │InputThread│
       │ (监控线程)    │ │(输入线程)  │
       └──────────────┘ └───────────┘
```

**核心设计理念**：
- 零CPU占用等待：使用事件等待，不占用CPU
- 快速响应：事件触发后立即响应，延迟 < 1ms
- 可扩展性：轻松添加更多事件

#### 模块职责划分
每个模块只做一件事：
- **Logger**：日志记录（多等级、彩色输出、线程安全）
- **InputModule**：用户输入处理
- **RestartModule**：进程监控和重启
- **ProcessManager**：进程管理（查找、启动、检查）
- **MQTTModule**：MQTT通信（可选功能）

## 性能优化方法论

### 问题诊断流程
1. **定位瓶颈**：通过日志和监控识别阻塞点
2. **根本原因分析**：多维度排查（文件I/O、连接池、内存、定时任务）
3. **分优先级修复**：高风险 → 中风险 → 低风险

### 常见优化手段
- **文件 I/O 异步化**：用 `fs/promises` 替代同步方法，性能提升 96%
- **HTTP 连接池优化**：增加连接池大小，添加清理机制，防止资源泄漏
- **定时任务批量处理**：用 `p-limit` 控制并发，避免串行阻塞
- **内存清理机制**：定期清理错误记录，防止内存泄漏
- **事件驱动替代轮询**：CPU占用从50%降到接近0%

### 性能验证
- 压力测试运行 4 小时以上
- 监控文件 I/O、HTTP 连接池、内存使用
- 日志分析阻塞点
- **量化对比**：优化前后数据对比表格

### 从 rsdocs 学到的性能优化

#### CPU 使用率优化对比

| 组件 | 轮询模式CPU占用 | 事件驱动CPU占用 | 优化效果 |
|------|----------------|----------------|---------|
| 主线程 | ~50-100% | ~0% | **降低99%+** |
| 监控线程 | ~5-10% | ~5-10% | 无变化 |
| 输入线程 | ~10-20% | ~5-10% | 降低50% |
| **总计** | **~65-130%** | **~10-20%** | **降低80-90%** |

## 安全设计原则

### 优先级：安全 > 效率 > 功能

#### 文件上传安全
- ZIP 文件解析使用纯 Buffer 操作，不转换为字符串
- 二进制签名搜索，避免编码问题
- Token 校验，防范 XXC 攻击

#### MQTT 鉴权架构
- **先鉴权后解析**：服务端先完成身份鉴权，仅合法消息才进入解析
- **加密帧头**：AES-ECB-128 加密 20 字节固定帧头
- **数据段纯二进制明文**：无加密开销，计算机原生解析
- **CRC16 完整性校验**：防传输篡改

#### 权限隔离
- 三层权限校验，防止权限提升
- 白名单强制（除主人指令外）
- 卡密唯一使用，用后即删

### 从 rsdocs 学到的安全思维

#### 安全分级系统
- **none**：基本连接权限
- **basic**：基础操作权限（状态查询等）
- **advanced**：高级操作权限（备份等）
- **admin**：管理员权限（重启、停止、CMD执行等）

#### 密钥管理原则
- 环境变量优先（推荐）
- 密钥文件次之
- 自动生成兜底
- 定期轮换密钥

#### 危险命令自动拦截
- 文件操作：`del`, `rm`, `rmdir`, `format`
- 系统控制：`shutdown`, `reboot`, `poweroff`, `halt`
- 进程管理：`kill`, `taskkill`, `tasklist`
- 用户管理：`net user`, `net localgroup`
- 系统工具：`reg`, `sc`, `schtasks`, `at`

## 经验知识库

### 文件处理
- ZIP 文件解析必须用纯 Buffer，`body.toString('binary')` 会损坏文件
- 二进制签名搜索用 `Buffer.from([0x50, 0x4B, 0x03, 0x04])`，不用 `indexOf('PK')`

### NapCat 插件
- 新插件默认禁用，必须 `setPluginStatus(pluginId, true)` 启用
- `loadDirectoryPlugin` 参数是目录名，不是完整路径
- API 返回 "No data returned" 可能实际已成功

### 群公告图片
- 不支持腾讯云临时 URL（缺少文件后缀）
- 正确方式：`get_image` API 获取本地路径 → 使用本地路径发送
- 支持：本地路径、公网 URL（标准格式）、base64 格式

### 性能优化
- 异步文件写入性能提升 96%
- HTTP Keep-Alive 第二次请求快 70%
- 定时任务并发控制避免串行阻塞
- **事件驱动替代轮询**：CPU占用降低 80-90%

### C++ 日志系统
- 多等级日志：DEBUG、INFO、WARN、ERROR、FATAL
- 线程安全：使用 CRITICAL_SECTION 互斥锁
- 彩色输出：根据日志等级使用不同控制台颜色
- 时间戳支持：格式 `%Y-%m-%d %H:%M:%S`

### MCBBS 签到工具
- PHP 实现，基于 Cookie 认证
- 随机签到语，支持自定义 rand.txt
- 定时执行（凌晨 4 点），失败自动重试

### MQTT 安全原则
- 统一主题结构设计
- 响应/状态/健康检查主题分离
- 鉴权优先于消息处理
